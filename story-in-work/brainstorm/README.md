# Brainstorm-Sandbox

Exploration ohne Canon-Commitment. Alles hier ist **nicht verbindlich**.

## Regel

- Nichts hier ist Canon
- `/brainstorm` schreibt hierher
- Von hier **keine** automatische Übernahme in `buch/`
- Canonisieren = bewusste Entscheidung via `/entwurf` oder manuelles Update

## Datei-Namensschema

```
brainstorm-{topic-slug}.md
brainstorm-{pov}-{topic-slug}.md   # bei POV-spezifischer Arbeit
```

## Tag-Konvention

- Untagged = Autor hat es gesagt
- `<AI>...</AI>` = KI-Vorschlag
- `<hidden>...</hidden>` = Autor-only, späterer Reveal
- `<rejected>...</rejected>` = verworfen mit Grund

## Aufräumen

Dateien die nach 30 Tagen nicht referenziert wurden: löschen oder nach `story-in-work/brainstorm/archiv/` verschieben.
